# Предсказание  рейтинга шоколада 

В рамках данного проекта разрабатывается и оптимизируется модель машинного обучения для автоматического прогнозирования рейтинга шоколада. Это позволяет производителям оценивать потенциал новых рецептур до запуска в производство, а дистрибьюторам — находить наиболее качественные партии товара.

Основная метрика качества: R²

#### Используемые инструменты и библиотеки
* **Анализ данных и визуализация:** `Pandas`, `NumPy`, `Seaborn`, `Matplotlib`, `association-metrics`
* **Масштабирование и кодирование:** `StandardScaler`, `MinMaxScaler`, `TargetEncoder`, `category-encoders`
* **Машинное обеспечение (Scikit-Learn):** `LinearRegression`, `RandomForestRegressor`, `SVR`, `GridSearchCV`, `Pipeline`, `CatBoostRegressor`, `Optuna`

#### Ход решения

### 1. EDA
Начальный этап включал в себя нормализацию текстовых полей и приведение признаков к нужным типам данных:
* Выполнено удаление скрытых пробелов в текстовых колонках, а пустые строки корректно заменены на значения `NaN`.
* Признак содержания какао успешно сконвертирован из строкового формата в числовой для корректного восприятия моделями.

### 2. Feature Engineering
Для извлечения скрытых зависимостей из текстовых описаний были созданы новые бинарные маркеры:
* `is_blend_type` — указывает, является ли шоколад смесью нескольких сортов какао-бобов.
* `is_blend_origin` — определяет, использовались ли при производстве бобы, импортированные из разных стран.
* `is_known_bean` — фиксирует, указан ли в спецификации конкретный сорт бобов или информация отсутствует.

### 3. Статистический анализ признаков
Для оценки структуры связей между признаками проведены следующие исследования:
* Рассчитана корреляционная матрица Пирсона для выявления линейных зависимостей между числовыми переменными.
* Построена матрица коэффициентов Крамера для измерения силы ассоциации между категориальными факторами.

### 4. Построение пайплайна
Для предотвращения утечки данных все этапы трансформаций были объединены в `Pipeline`:
* Категориальные признаки закодированы через `TargetEncoder`, при этом параметр сглаживания оптимально подбирался с помощью `GridSearchCV`.
* Числовые признаки масштабированы с использованием `StandardScaler` и `MinMaxScaler`.
* Базовая модель линейной регрессии показала на тесте скромный результат **R² ~ 0.30**.

### 5. Обучение ансамблей и оптимизация гиперпараметров
Для максимизации предсказательной способности были протестированы нелинейные алгоритмы и ансамбли:
* Проведено сравнение моделей `RandomForestRegressor` и `SVR`. Оптимизация случайного леса через **Optuna** позволила поднять качество до **R² ~ 0.42**.
* **CatBoost:** Наилучший результат продемонстрировал градиентный бустинг от Яндекс. Использование нативного режима с передачей списка категориальных переменных в `cat_features` и последующая настройка через **Optuna** привели к итоговому значению **R² ~ 0.45**.

## Финальные результаты
LinearRegression: 0.3
RandomForestRegressor + Optuna: 0.42
CatBoostRegressor: 0.45

## Выводы
1. Качественная очистка текстовых полей и выделение флагов позволили моделям лучше улавливать элитность и происхождение сырья.
2. Использование нативного кодирования категориальных признаков в CatBoost показало явное преимущество над ручным применением `TargetEncoder` в Pipeline, обеспечив наивысший коэффициент детерминации на тестовой выборке.
